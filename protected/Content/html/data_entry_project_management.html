Data Entry and Project Management

Overview


Data Entry and Project Management is now integrated into the DataPortal. The ability to perform functions related to these concepts are governed by the user holding a specific "Role" within the DataPortal. These roles form a hierachy as follows:

user -> projectUser -> projectDataEntry -> hubAdmin -> superAdmin

On sign-up, the default role is "user". This role can only search "globally visible" surveys and  records, a concept defined further below.

If a user is part of a research project, they can be added to a project, and so have the "projectUser" role. This increases the number of surveys and records visible to them to include "privately visible" records owned by a project they are a member of.

If a user needs to be able to add new data to the DataPortal, they need the "projectDataEntry" role. This allows new Data Entry tabs to be visible within the DataPortal.

The "hubAdmin" role allows access to the Project Management tab, which is used to assign surveys to projects, changing a surveys visibility, as well as adding users to projects.

Users with the role of "superAdmin" can change other users roles.

Each of these roles supercedes that of the one before it, so the "hubAdmin" users are also able to perform data entry tasks.

User Guide


Below is a picture of the DataPortal once a user is logged in. This view is the same whatever role the user holds.


As in previous versions there is a button in the top right of the screen which provides access to the "My Account" window.

Clicking that button shows the window below. The "ian" user  has the standard default  "user" role.  There is only the standard 5 tabs visible, mostly inactive or in early development stages.



If the user has a the role of "projectDataEntry" or higher, when the "My Account" button is pressed the My Account window will show the extra tabs "Survey Entry" and "Tagging", shown below.



The "Survey Entry" tab is used for adding new Surveys to the DataPortal Database. Surveys can also be loaded, updated and new Questions and Responses can be added here.



The "Tagging" tab is used to add Qualitative data to the DataPortal DataBase.



If the user has the role of "hubAdmin" or higher, the "Project Management" tab becomes visible.
